## Раздел «Помощь» — план, структура и оглавление

Полноценный справочный центр внутри приложения ТопФокус: единая точка входа для пользователя, объясняющая методологию, все модули, тарифы, безопасность и партнёрские программы. Раздел встраивается как отдельная страница `/help` с боковым/верхним оглавлением, поиском по статьям, хлебными крошками и глубокими ссылками на конкретные функции из других экранов (иконка «?»).

---

### 1. Архитектура раздела

```
/help
├── Hero + поиск + быстрые ссылки (Старт, FAQ, Связаться)
├── Sidebar/TOC (sticky, 3 уровня вложенности)
├── Контент статьи (MDX/JSON-структура)
│   ├── Краткое описание (TL;DR)
│   ├── Пошаговая инструкция со скриншотами
│   ├── «Хитрости и лайфхаки»
│   ├── Связанные статьи
│   └── Блок «Это помогло?» (фидбек)
└── Footer: версия приложения, ссылка на правовые документы
```

Технически: `src/pages/Help.tsx` + `src/components/help/*` (HelpSidebar, HelpArticle, HelpSearch, HelpBreadcrumbs, HelpFeedback). Контент — массив статей в `src/content/help/{ru,en}/*.mdx` или JSON со схемой `{ id, slug, title, category, tags, body, related[] }`. Поиск — клиентский Fuse.js по title/tags/headings. Поддержка RU/EN через `LanguageContext`. Иконка «?» в `AppHeader` каждого модуля ведёт на якорь `/help#<slug>`.

### 4. План работ (этапы реализации после утверждения)

1. **Каркас (1 коммит):** `Help.tsx`, роут `/help`, sidebar+search+breadcrumbs+article-shell, JSON/MDX-схема контента, перевод RU/EN.
2. **Контент-движок:** компонент `<HelpArticle>`, рендер MDX/JSON, оглавление статьи (H2/H3), якоря, связанные статьи, фидбек-кнопки.
3. **Поиск:** Fuse.js по `title`+`tags`+`headings`, подсветка совпадений, история поиска в localStorage.
4. **Интеграция «?»:** в `AppHeader` и ключевых диалогах добавить кнопку справки → `/help#slug`.
5. **Наполнение по разделам** (по 2–3 раздела за итерацию, ~95 статей): сначала «Начало работы», «Методология», «Дашборд», «Задачи», «Цели» → затем остальные по приоритету частоты обращений.
6. **Скриншоты:** генерация в `src/assets/help/*.webp` с тёмной/светлой темой; lazy-load.
7. **Аналитика:** трекинг просмотров статей и негативного фидбека для последующего улучшения.
8. **QA:** проверка RU/EN, ссылок, мобильной вёрстки, доступности (a11y), глубоких ссылок.

### 5. Технические детали

- **Хранилище контента:** статичные MDX-файлы (компилируются на билде) либо JSON в `src/content/help/`. Рекомендуется MDX — позволяет вставлять интерактивные демо-компоненты (мини-Pomodoro, демо-кольца).
- **i18n:** ключ статьи одинаковый, файлы `ru.mdx` / `en.mdx`; язык берётся из `LanguageContext`.
- **SEO:** одна `<h1>` на статью, мета-title/description из frontmatter, canonical, JSON-LD `Article` + `BreadcrumbList`, `lazy` на картинках.
- **Дизайн-токены:** только семантические (`bg-card`, `text-foreground`, `border-border`, акценты модулей: `task`, `habit`, `finance`, `service`, `team`). Радиусы — `var(--radius-card)`.
- **Доступность:** клавиатурная навигация по TOC, `aria-current` для активной статьи, контраст AA, фокус-видимый.
- **Производительность:** code-split страницы Help, prefetch при ховере на пункт меню, изображения `.webp` с placeholder-blur.
- **Память проекта:** соблюдены ограничения — APP_URL, прокси-маршруты, AppHeader-правила, радиусы, цветовая семантика модулей.
